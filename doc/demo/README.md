# 財報×永續整合儀表板（Demo）

## 目的
本 Demo 用於展示「金融資訊 PM / 商業分析師」的端到端能力：
1) 需求拆解（商業需求文件 BRD、功能需求文件 FRD）
2) 資料源盤點與資料契約（資料源覆蓋矩陣）
3) 資料治理與核決（資料放行、證據包、版本回溯）
4) 驗收與測試（驗收準則、測試案例、需求追蹤矩陣 RTM）
5) 儀表板 Demo 設計（畫面、指標、互動、例外處理）

## Demo 範圍（本次做得到、且資料源可支持）
- 財報：營收、毛利率、營益率、淨利率、ROE、負債比、流動比、營運現金流、自由現金流
- ESG：職災人數/比率（職業安全衛生）、（可擴充）資訊安全、治理結構、溫室氣體等揭露彙總資料
- 治理：每次資料更新需生成「證據包」並完成「核決」，儀表板只讀取已放行版本；支持版本回溯比較

> 資料源：TWSE OpenAPI（財報/ESG 彙總）、ESG 數位平台匯出 Excel、公司永續報告書 PDF（補充證據）

## 5 分鐘面試展示腳本（建議照念）
1) 我先說明資料源與資料契約：哪些指標來自 TWSE OpenAPI、哪些來自 ESG 平台匯出
2) 我展示「資料放行」：每次更新都要有證據包、指紋、核決紀錄，避免數字被質疑
3) 我展示儀表板：財報趨勢、ESG 指標趨勢、交叉指標（例如：營收 vs 職災率）
4) 我展示「版本回溯」：選擇上一版放行資料，比對差異，並指出差異原因（資料源更新或更正）
5) 我展示 RTM：每個需求都對應到驗收與測試，確保不是「做漂亮而已」
