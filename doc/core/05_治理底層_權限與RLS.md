# 治理底層｜權限與 RLS（Row Level Security）

本文件定義角色、最小權限與資料可見範圍，確保「能看的人看到、能改的人只能新增版本」。

## 1. 角色定義
- 送件者：提供資料來源（系統或人）
- 分析者：查看分析結果與追溯摘要
- 審核者：做 G5 放行/拒絕核決
- 稽核者：只讀查詢與匯出
- 管理者：授權與維運（仍不可覆寫既有版本）

## 2. 最小權限矩陣（摘要）
- Raw：任何人不可改寫；只有系統/送件者可新增
- Curated：只能新增版本；不得更新舊版
- Mart：只能新增版本；不得更新舊版
- 核決事件：只能新增；不得更新/刪除
- 發佈標記：只能指向已放行版本；保留歷史紀錄

## 3. RLS 規則（概念）
- 分析者：可讀取「已發佈版本」與其追溯摘要；不可讀取未放行版本
- 審核者：可讀取待核決版本與 DQ 結果；可新增核決事件
- 稽核者：可讀取所有版本與核決事件；不可新增任何資料
- 管理者：可管理角色授權；不可更新不可竄改內容（指紋/版本/歷程）

> 實作層（資料庫政策）不在本文展開；本文作為需求與稽核依據。
