# 治理底層｜證據包與稽核回放

本文件定義「證據包」與「稽核回放」：任何對外展示或輸出的數字，都必須能回推來源與核決紀錄。

## 1. 證據包（Evidence Pack）最小內容
任何可被引用的輸出（含 Dashboard 數字、報表欄位、圖表點位）都必須具備：

1) 來源（Source）
- 財報：來源平台/檔案識別/期間
- 永續：報告書年度＋頁碼/章節（或摘錄文字）
2) 指紋（Fingerprint）
- 對來源內容或摘要產生內容指紋
3) 版本（Version）
- Raw 版本、Curated 版本、Mart 版本、公式版本
4) 核決（Approval）
- 經過哪些 Gate
- 哪個核決事件放行
- 放行理由與附件（如有）

## 2. 稽核回放（Audit Replay）能力清單
稽核回放最小能力：
- AR-01 從任一 Dashboard 數字，查到對應指標版本（Mart）
- AR-02 從指標版本，查到使用的 Curated 版本與公式版本
- AR-03 從 Curated 版本，查到 Raw 來源引用與指紋
- AR-04 查到核決事件（人/時/事/因）與 Gate 紀錄
- AR-05 可匯出摘要（供抽查）

## 3. 缺證據的處置
- 缺來源引用：禁止進入發佈 Gate
- 缺指紋：禁止發佈
- 缺核決理由：禁止發佈
- 缺版本：視為不合格資料，退回補件
