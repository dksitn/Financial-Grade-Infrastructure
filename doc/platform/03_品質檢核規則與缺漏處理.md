# 品質檢核規則與缺漏處理

本文件定義 G3 的品質檢核（DQ）最小集合，確保資料能被合理比較，不因缺漏或口徑錯誤造成誤導。

## 1. 檢核類型（最小集合）
- DQ-01 完整性：必要欄位是否齊備（來源引用、單位、期間）
- DQ-02 一致性：同一資料集幣別/單位/期間是否一致
- DQ-03 缺漏：同公司同期間必要科目/必要指標是否缺
- DQ-04 異常跳動：比率/指標是否超過門檻（只提示，不直接下結論）
- DQ-05 多版本：同期間多版本是否有更正原因與版本說明

## 2. 輸出格式（DQ 結果）
- result：通過/警告/失敗
- issues：缺漏清單、異常清單、口徑衝突清單
- recommendation：補件/修正映射/需人工核對

## 3. 缺漏策略（不阻擋 vs 阻擋）
- 阻擋：缺來源引用、單位不可辨識、期間不明
- 不阻擋但提示：缺部分科目/指標（以「缺」呈現），需審核者決定是否放行發佈
