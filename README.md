# 文件總覽（放 GitHub 用）

本資料夾是一套「治理底層＋資料中台＋分析功能」的端到端文件包，目標是讓讀者（面試官/專案利害關係人）能快速看懂：

- 核心治理底層如何確保：不可竄改、可追溯、可稽核、可回滾
- 資料中台如何分層：Raw → Curated → Mart → Service → Dashboard
- 兩個實際功能案例（財報分析、永續分析）如何落在同一套治理底座上
- 每個案例都有可驗收交付物：BRD / FRD / 流程 / 驗收 / 測試 / RTM

## 建議閱讀順序
1) core/01_共同架構總覽  
2) core/02~05（治理底層：資料核決/證據包/Gate/RLS）  
3) platform/01~06（資料來源、分層、資料契約、品質檢核、指標、Plugin、Dashboard）  
4) cases/financial（財報分析全套）  
5) cases/sustainability（永續分析全套）

## 文件一致性原則（很重要）
- Case 文件不重複寫治理規則；一律引用 core/ 與 platform/ 的規格
- 所有需求、驗收、測試、RTM 都用同一套編碼方式，確保可追溯
- 缺資料「顯示缺」不補 0；更正採「新增版本」不覆寫
